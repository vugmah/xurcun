# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`xurcun.az` — the public site + admin CMS for **Xurcun Chain Of Boutiques** (premium dried fruit, nuts, sweets, gift boxes; Baku). Single repo holds a React SPA **and** its Hono API server. Public surface is a 5-language (AZ/RU/EN/TR/AR) brand/catalog/QR-menu site; `/admin/*` is a separate CMS.

## Commands

```bash
npm run dev          # Vite dev server on :3000 (serves SPA + Hono API via @hono/vite-dev-server)
npm run build        # vite build + esbuild-bundle api/boot.ts → dist/boot.js
npm start            # production: NODE_ENV=production node dist/boot.js
npm run check        # tsc -b (type check; the real "does it compile" gate)
npm run lint         # eslint .
npm test             # vitest run
npx vitest run path/to/file.test.ts     # single test file
npx vitest run -t "name"                # single test by name
npm run db:generate | db:migrate | db:push   # drizzle-kit (schema in db/, migrations gitignored)
```

`build` won't run without **`ADMIN_SECRET_KEY`** set at runtime (see Gotchas). `prebuild` runs `scripts/check-null-bytes.cjs` — if build fails there, a source file has stray null bytes.

## Architecture (the parts that span files)

- **One server, two roles.** `api/boot.ts` is the entry: a Hono app that mounts tRPC (`/api/trpc/*`), file uploads, and `robots.txt` / `sitemap.xml` **routes**, then serves the built SPA. In dev it runs through `@hono/vite-dev-server`; in prod it's esbuild-bundled to `dist/boot.js`. Static serving + SPA fallback live in `api/lib/vite.ts` (`serveStaticFiles`): unknown paths return `index.html` (200) **only when `Accept: text/html`**, else JSON 404.
- **Client ↔ server is tRPC** (`src/providers/trpc.tsx`, `@trpc/react-query`). DB is **Drizzle ORM + MySQL** (Railway). Don't hand-write fetch calls for app data; add a tRPC procedure.
- **Routing is `BrowserRouter`** (`src/main.tsx`, routes in `src/App.tsx`). It was migrated from HashRouter — `main.tsx` keeps a shim that rewrites legacy `#/…` links to real paths. Catch-all `path="*"` → `NotFoundPage` (sets `noindex` to avoid SPA soft-404s). When adding routes, both the client route and the server SPA-fallback must cover them.
- **Two visual systems.** Public pages use the cream/gold `.xc` brand theme; `/admin/*` uses a dark CMS theme. Brand tokens + shared primitives (`.reveal`, `.tag`, `.ornament`) are in `src/xurcun-base.css`; page styles in `xurcun-home.css`, `xurcun-menu.css`, `xurcun-catalog.css`. **`design-system/MASTER.md` is the brand source of truth** (colors, fonts, contrast rules) — read it before changing visuals.
- **i18n** (`src/lib/LanguageContext.tsx`): 5 languages via a `?lang=` query param (crawlable), persisted to localStorage; `getInitialLang()` prefers the URL param. Arabic flips `<html dir>` to RTL. Strings in `src/lib/translations.ts`; page-level copy often lives inline as `Record<Lang, string>` maps.
- **SEO is split.** Static `<head>` (Organization/WebSite/Store JSON-LD, hreflang for 5 langs via `?lang=`, OG/Twitter) is in `index.html`. Per-page meta is `src/sections/SEO.tsx` via `react-helmet-async`. `robots.txt`/`sitemap.xml` are generated by **`api/boot.ts` routes** and win over the (stale) `public/` equivalents.
- **Reveal-on-scroll**: `.reveal` starts at `opacity:0`; `HomePage.tsx` adds `.in` via IntersectionObserver with a 1.5s safety net, and `xurcun-base.css` has a `@media(scripting:none)` fallback so no-JS crawlers still see content. Don't gate real content behind a JS-only reveal without an equivalent fallback.

## Gotchas

- **`ADMIN_SECRET_KEY`** (`api/middleware.ts`) is required — the server throws on startup without it. Set it in `.env` locally and in Railway env vars.
- **Deploy = Railway, fronted by Cloudflare.** Static assets (e.g. `/assets/favicon.svg`) are Cloudflare-cached (`max-age=14400`); after changing them, purge Cloudflare cache or the old version persists. `curl` to the live domain hits Cloudflare **Bot Fight Mode** (403 challenge) — verify redirects/headers in a real browser, not curl.
- Canonical domain is **xurcun.az**; `xurcun.com` 301-redirects to it (Cloudflare page rule). Keep canonical/hreflang/sitemap pointing at xurcun.az.
- `.env` is gitignored; `db/migrations/*.sql` is gitignored.

## Conventions

- TypeScript strict; path alias `@/` → `src/`.
- Match the surrounding file's style. Public UI is hand-written CSS in the `.xc` cascade (not Tailwind utilities); admin uses Tailwind + shadcn/ui (`src/components/ui/`).
- Accessibility is load-bearing here: keep `:focus-visible` rings, ≥44px touch targets, `prefers-reduced-motion` fallbacks, and `@media(hover:hover)` guards on hover effects (esp. the touch-first QR menu).
